几个css样式

By 小韦德

Published 2017-08-09

##box-sizing

由于各种浏览器有各自的盒模型，有自己的标准，所以如果设置margin,padding他们所加在的位置也会不同，这样看到的宽高可能就会不一样了。这样我们有box-sizing这个属性可以设置，它有两个值content-box和border-box，如果box-sizing: content-box，那么元素设置宽高后，加padding,margin，还是会按照浏览器标准的不同而解析；设置box-sizing: border-box 元素宽高在各种浏览器则会一致，不会因padding,border而改变。
一些专家甚至建议所有的WEB开发者：将所有的元素的box-sizing都设为border-box

##flex

如果不知道元素高度，那么垂直居中就不容易实现，所以我们可以display: flex，然后设置align-items: center就可以了，其他几个属性：
flex-direction: row|row-reverse|column|column-reverse 设置元素的排列顺序
flex-wrap: nowrap|wrap|wrap-reverse  设置一行排不下如何换行
justify-content: flex-start|flex-end|center|space-between|space-around  x轴的对齐方式
align-items: flex-start|flex-end|center|baseline|stretch  y轴的对齐方式

vue初体验(一)

By 小韦德

Published 2017-08-02

##快速的搭建一个脚手架

根据vue官方文档，只需在命令行 npm init webpack '项目名' 即可，里面配置信息都已配置好

##安装所需依赖

在根目录package.json文件下，dependencies对象写输入所需的依赖及版本号，npm install即可安装依赖；或者在命令行npm install 依赖 --save也可安装所需依赖

##项目结构

所有的页面都卸载src目录下，所以我们主要工作的地方就是在src目录下面；build，config目录为打包跟配置信息，一般我们不会更改,node_modules为安装的依赖。

##页面间相互调用

在src目录下写我们的组件，A调用B，我们可以在B中 export 或者 export default，当然两种的写法在A中也是不一样的调用写法,export 调用为 import {名} from B，export default调用为 import 名 from B
这样，我们可以在build下的webpack.base.conf.js下面配置信息，resolve对象下的aias配置，以此来简化调用来源

git操作

By 小韦德

Published 2017-07-25

##几个常用的git操作命令

比较容易且经常用到的操作

git add. 添加所有更改到本地
git commit -m ''  给此次添加的代码一个描述
git status 查看一下当前本地git的状态，添加删除更改都可以看到
git push origin master  把本地代码提交到远程分支 
git pull  拉取代码
git checkout -b new-master  新建一个new-master本地分支

合并代码

git merge
假设我本地跟远程各有一个代码库，且都有两个分支：master和dev分支，master跟dev一一对应。如果我在dev更改了许多内容并且提交到了服务器，现在也想在master加上同样的内容，就可以用到：
git checkout master  切换到master分支
git merge dev 合并

代码回滚

git reset --hard commit-id  回滚到commit-id，commit-id之后的提交都会删除
git reset --hard HEAD~5  删除最近5次的提交
git-log  可以打印出最近的操作，找到commit-id

远程代码回滚

如果线上发现bug，需要回滚到上个版本，更改后再重新发布，可以先在本地回滚到上一个版本，回滚如上，删除远程分支，重新push本地代码
git reset --hard commit-id  本地回滚到commit-id
git push origin :master  删除远程master分支
git push origin master  把回滚后的代码推到远程master分支

console的几种用法

By 小韦德

Published 2017-07-13

##console之数字

输入： console.log("parse": %d, floating: %f, 1, 1.5);
输出： parse: 1, floating: 1.500000

%i && %d 代表整数，%f代表浮点数

##console之自定义

const success = [
    "background": "green",
    "color": "#fff",
    "display": "block",
    "text-align": "center"
].join(";");
console.info("this is success", success);

这样自定义了一种success样式，打印出来就是对应的样式。

##console之断言

let flag = false;
console.assert(flag, "显示");
flag = true;
console.assert(flag, "不显示");

assert()：第一个参数为布尔值，入为false则输出到控制台，同时提示红色错误信息；为true则不显示

##console之检查对象

console.dir(document.body);
输出： body对象的所有属性

如果需要检查一个对象，用dir这种结构化方式显示的话可以简洁。

##console之表格

console.table（['Javascript'，'PHP'，'Perl'，'C ++']）;
输出一个table布局的列表
